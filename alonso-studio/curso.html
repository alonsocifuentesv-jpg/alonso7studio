Módulo 01

Cómo funciona
el mundo del diseño web

Antes de tocar código, entiende el negocio. Qué se vende, quién lo compra y cómo se gana plata de verdad.

🌐
Qué es una página web en términos simples

Una página web es un archivo de texto con instrucciones especiales. Cuando alguien entra a tu sitio, su navegador (Chrome, Safari) lee ese archivo y lo convierte en lo visual que ves en pantalla.

No hay magia. Es texto que el navegador interpreta.

Analogía: Una página web es como una receta de cocina. El HTML es la lista de ingredientes, el CSS es cómo presentar el plato, y el JavaScript es el cocinero que hace cosas en tiempo real.
💡
Qué diferencia a un buen diseñador web de uno malo
❌ El malo

Usa templates, copia código sin entender, entrega tarde, no explica qué hizo, no puede hacer cambios solo.

✅ El bueno

Entiende qué vende el cliente, diseña con criterio, entrega rápido, puede explicar cada decisión y hacer ajustes al momento.

Los que ganan bien no son necesariamente los que saben más código. Son los que entienden el negocio del cliente mejor que nadie.

Módulo 02

HTML — La estructura
de todo sitio web

HTML no es programar. Es como escribir un documento de Word pero con etiquetas que le dicen al navegador qué es cada cosa.

📄
Qué es HTML

HTML significa HyperText Markup Language. Es el esqueleto de toda página web. Sin HTML no existe nada.

Funciona con etiquetas que abren y cierran:

<!-- Esto es una etiqueta que abre y cierra --> <h1>Soy un título grande</h1> <p>Soy un párrafo de texto</p> <a href="https://...">Soy un link</a> <img src="foto.jpg"/> <!-- Las imágenes no necesitan cierre -->

Cómo leer el HTML de cualquier página

Abre Chrome, entra a cualquier página y presiona F12. Ves todo el HTML real. Esto es lo que debes hacer con las páginas que admires para entender cómo están hechas.

Módulo 03

CSS — El diseño
y los colores

CSS es lo que hace que todo se vea bien. Sin CSS, una página es texto negro sobre fondo blanco. CSS es tu herramienta de diseño.

🎨
Cómo funciona CSS

CSS selecciona un elemento HTML y le aplica estilos. La lógica es siempre la misma:

/* Selecciono el elemento → le pongo propiedades */ h1 { color: #c9a84c; /* color del texto */ font-size: 3rem; /* tamaño del texto */ font-weight: 700; /* grosor (700 = negrita) */ text-align: center; /* alineación */ } .mi-boton { background: #0d0d0d; /* color de fondo */ color: white; /* color del texto */ padding: 14px 32px; /* espacio interno */ border-radius: 0px; /* bordes redondeados */ }
El punto (.) significa clase. En HTML pones class="mi-boton" y en CSS escribes .mi-boton para darle estilos. Así conectas HTML con CSS.
📱
Cómo funciona el diseño responsivo (mobile)

Para que un sitio se vea bien en celular, usamos media queries. Es una instrucción que dice "cuando la pantalla sea pequeña, aplica estos estilos en cambio":

/* Estilo normal (computador) */ .titulo { font-size: 4rem; } /* Cuando la pantalla es menor a 768px (celular) */ @media(max-width: 768px) { .titulo { font-size: 2rem; /* más pequeño en celular */ } }

Módulo 04

JavaScript — Lo que
hace que todo se mueva

JS es el único de los tres que realmente programa. Hace que los botones hagan cosas, los menús se abran, los formularios funcionen.

⚡
Qué hace JavaScript que HTML y CSS no pueden
  • Abrir y cerrar el menú hamburguesa en celular
  • Validar que el formulario esté completo antes de enviarlo
  • Animar elementos cuando el usuario hace scroll
  • Mostrar y ocultar secciones (FAQ, tabs, modales)
  • Enviar datos al WhatsApp con el texto pre-armado
  • El carrito de compras de la tienda
  • El calendario de agenda con horarios
Regla simple: Si algo "hace algo" cuando el usuario interactúa, es JavaScript. Si solo se ve, es HTML y CSS.
🔧
Conceptos básicos que debes reconocer
// Variable — guardar un valor const nombre = "Alonso"; // Función — una acción que se ejecuta cuando la llamas function enviarFormulario() { // aquí va la lógica } // Seleccionar un elemento HTML desde JS const boton = document.getElementById('mi-boton'); // Hacer algo cuando el usuario hace clic boton.addEventListener('click', function() { // esto se ejecuta al hacer clic });

No necesitas memorizar esto. Necesitas reconocerlo cuando lo ves en el código para entender qué hace.

📋
El JS de tu página explicado

En tu script.js hay 5 funciones principales:

  • Cursor personalizado — mueve el punto dorado que sigue el mouse
  • Reveal on scroll — hace aparecer elementos con animación al bajar
  • seleccionarPlan() — cuando alguien hace clic en un plan, lo pre-selecciona en el formulario
  • toggleFaq() — abre y cierra las preguntas frecuentes

Módulo 05

Estructura de archivos
como un profesional

Cómo organizar los archivos de un proyecto real para que ningún desarrollador pueda cuestionarte.

📁
La estructura mínima profesional
mi-proyecto/ │ ├── index.html # La página principal ├── styles.css # Todo el CSS separado ├── script.js # Todo el JavaScript separado │ ├── assets/ │ ├── img/ # Todas las imágenes │ │ ├── logo.png │ │ └── hero.jpg │ └── fonts/ # Tipografías locales (si las hay) │ └── pages/ # Páginas adicionales ├── servicios.html └── contacto.html
Regla de oro: Un archivo hace una sola cosa. HTML = estructura. CSS = diseño. JS = comportamiento. Nunca mezclar.
🔗
Cómo se conectan los archivos

En el <head> del HTML enlazas el CSS:

<link rel="stylesheet" href="styles.css"/>

Antes del </body> enlazas el JS:

<script src="script.js"></script>

Así de simple. El HTML carga el CSS y el JS como archivos externos.

Módulo 06

Cómo usar la IA
sin que se note

La IA es una herramienta, no el diseñador. Así la usas para que el resultado sea tuyo y nadie pueda cuestionarte.

🧠
El cambio de mentalidad

La diferencia entre alguien que "usa IA" y alguien que "trabaja con IA" es quién tiene el criterio.

❌ Usar IA mal

"Hazme una página web para una barbería" → copiar y pegar lo que salga

✅ Usar IA bien

Tú defines estructura, colores, secciones, textos. La IA ejecuta lo técnico. Tú revisas y corriges.

📋
El flujo de trabajo correcto
  • Paso 1 — Tú diseñas en papel: Qué secciones va a tener, en qué orden, qué colores, qué estilo
  • Paso 2 — Tú escribes los textos: Los textos reales del cliente, no inventados por IA
  • Paso 3 — Le pides el código a la IA: Con instrucciones específicas, no genéricas
  • Paso 4 — Tú revisas el código: Que esté separado, sin comentarios, con tu lógica
  • Paso 5 — Tú ajustas: Cambias colores, espaciados, textos según tu criterio
  • Paso 6 — Tú presentas: El resultado tiene tu sello, no el de la IA
💬
Cómo pedirle código a la IA correctamente
Prompt profesional: "Crea el archivo styles.css para una barbería de lujo. Paleta: negro #0d0d0d y dorado #c9a84c. Tipografía Inter. Sin comentarios. Sin estilos inline. Incluye: nav fijo, hero con grid 2 columnas, sección de servicios con 3 cards, y footer. Mobile responsive con media query a 768px."

Mientras más específico seas tú, mejor y más profesional sale el resultado. La IA no tiene criterio propio — tú se lo das.

Cómo trabajar
con clientes reales

El código es el 30% del trabajo. El 70% es saber relacionarte con el cliente para que pague, quede contento y te recomiende.

🤝
La primera reunión con un cliente
  • No vendas en la primera reunión. Pregunta y escucha. ¿Qué necesita? ¿Qué problema tiene? ¿Cuántos clientes quiere generar?
  • Preguntas clave: ¿Tienes fotos del local? ¿Tienes logo? ¿Cuáles son tus servicios y precios? ¿Qué redes sociales tienes?
  • Al final: "Déjame preparar una propuesta y te la mando hoy mismo"

Cómo manejar los cambios

Todo cliente va a pedir cambios. Así lo manejas sin perder plata ni tiempo:

  • Cambios grandes (nueva sección, rediseño) son precio aparte
  • Siempre confirma por escrito qué se cambió y cuándo

Módulo 09

Cómo cobrar
lo que realmente vales

El precio es una señal de valor. Cobrar muy barato te perjudica más de lo que te ayuda.

💰
Por qué cobrar barato es un error
  • Los clientes baratos son los más exigentes y los que más problemas dan
  • Precio bajo = percepción de baja calidad en la mente del cliente
  • Si cobras $50.000, necesitas 10 clientes para ganar lo mismo que con 1 a $500.000
  • El cliente que paga bien respeta tu trabajo y te recomienda mejor
📊
Cuándo y cómo subir tus precios
CuándoPrecio sugerido
Hoy — 0 clientes reales$150.000 – $250.000
Con 3 proyectos reales terminados$250.000 – $400.000
Con 8+ proyectos y reseñas reales$400.000 – $700.000
Con especialización y proceso claro$700.000 – $1.200.000
Regla: Cuando el 80% de los prospectos acepta tu precio sin negociar, es momento de subirlo.
🤝
Cómo presentar el precio para que no lo cuestionen
  • Siempre muestra el demo antes de mencionar el precio
  • Compara con lo que cobra una agencia ("una agencia cobra $580.000, yo $250.000")
  • Separa en cuotas: "50% ahora, 50% cuando veas el resultado"
  • Nunca justifiques el precio ni lo bajes antes de que lo pidan
  • Si lo piden, pregunta primero: "¿Qué presupuesto tienes en mente?"
